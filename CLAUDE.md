# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm run dev            # Next.js dev server (Turbopack) on :3000
npm run build          # Production build
npm run start          # Run the built app
npm run lint           # ESLint via next/core-web-vitals + next/typescript
npm run slicemachine   # Start Slice Machine UI on :9999 for editing Prismic models
```

There is no test runner configured. TypeScript path alias `@/*` maps to `./src/*`.

## Required environment variables

- `NEXT_PUBLIC_PRISMIC_ENVIRONMENT` — overrides the Prismic repository name from `slicemachine.config.json` (used to point dev/preview at a non-default Prismic env).
- `STRIPE_SECRET_KEY` — used by `src/app/api/checkout/[uid]/route.ts` and `src/app/success/page.tsx`. Stripe API version is pinned to `2025-07-30.basil`.

## Architecture

### Prismic-driven content pipeline
The site renders almost entirely from Prismic content via `SliceZone`. Three custom types live in `customtypes/`:
- `homepage` (single) — its `slices` Slice Zone composes the landing page
- `product` (repeatable, has UID, name, price *in cents*, image, description) — drives `/api/checkout/[uid]`
- `switch` (repeatable, name + hex color) — referenced via content relationships from the SwitchPlayground slice

Generated TypeScript types for all custom types live in `prismicio-types.d.ts` (regenerated by Slice Machine — do not hand-edit).

`src/prismicio.ts` builds the Prismic client. In production it sets `cache: "force-cache"` with the `"prismic"` tag; the `/api/revalidate` POST endpoint calls `revalidateTag("prismic")` so a Prismic webhook can flush the cache. In development, `next.revalidate: 5` keeps content fresh without a webhook. `enableAutoPreviews` is wired in here, and `<PrismicPreview>` is rendered in `src/app/layout.tsx`. Preview entry/exit go through `/api/preview` and `/api/exit-preview`.

### Slices
Each slice lives in `src/slices/<Name>/` with `index.tsx`, `model.json`, `mocks.json`, and a screenshot. `src/slices/index.ts` is **auto-generated by Slice Machine — do not edit**. To add a slice, use Slice Machine; never register it manually here.

Current slices: `Hero`, `BentoBox`, `ColorChanger`, `Marquee`, `PurchaseButton`, `SwitchPlayground`. Hero and ColorChanger each have a sibling `Scene.tsx` for their R3F scene.

The Slice Simulator is mounted at `/slice-simulator` and is referenced by `slicemachine.config.json` (`localSliceSimulatorURL`).

### 3D scenes (React Three Fiber + Drei + GSAP)
`src/components/Keyboard.tsx` loads `public/keyboard.gltf` (a Vapor75 mesh with one `K_*` mesh per key) via `useGLTF`. The component `forwardRef`s a `KeyboardRefs` shape exposing per-row switch groups and per-key keycap meshes — this is what the Hero scroll timeline uses to drive the column-by-column wave animation.

`src/components/Keycap.tsx`, `src/components/Switch.tsx` (with `SOUND_MAP` and audio in `public/sounds/`) are reused across multiple slices.

`KEYCAP_TEXTURES` is the canonical list of keycap colorways. It is **defined and exported from `src/slices/ColorChanger/index.tsx`** and re-imported by its `Scene.tsx`. If you add a colorway, add the matching `*_uv.png` file to `public/`.

### Animation conventions
- All GSAP work is wrapped in `useGSAP` and gated by `gsap.matchMedia("(prefers-reduced-motion: no-preference)")` with a `reduce` branch that sets the final visual state immediately. Preserve this pattern when adding new animated elements.
- Plugins (`SplitText`, `ScrollTrigger`, `useGSAP`) are registered at the top of each consuming module — register them locally rather than relying on a global setup.
- The Hero scene briefly sets `document.body.style.overflow = "hidden"` during the intro timeline and clears it on completion. If you change the intro, ensure overflow is always restored or scroll-locking will leak.

### Checkout flow
1. Client calls `checkout()` from `src/checkout.ts`, which POSTs to `/api/checkout/vapor75` (the product UID is currently hardcoded there).
2. `src/app/api/checkout/[uid]/route.ts` fetches the Prismic `product` document by UID, builds a Stripe Checkout Session with `price.unit_amount = product.data.price` (price is stored *in cents* in Prismic), and returns `{ url }`.
3. Client redirects to Stripe; on success Stripe sends the user to `/success?session_id=...`, which retrieves the session server-side and renders order confirmation.

### Layout & styling
- Tailwind v4 via `@tailwindcss/postcss`; theme tokens (including the `font-black-slanted` / `font-bold-slanted` Roboto Flex variation-settings presets) are declared in `src/app/globals.css` under `@theme inline`.
- `src/components/Bounded.tsx` is the standard page section wrapper — use it for new content sections to keep horizontal padding and `max-w-7xl` consistent.
- Prettier is configured with `prettier-plugin-tailwindcss` for class sorting.
